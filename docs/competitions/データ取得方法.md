## 結論

調べるべき「二つ」を **ROGII** と **Predicting Stellar Class / S6E6** として整理します。

**推奨データ取得方法は2系統です。**

| 用途                                 | 方法                                           |        優先度 |
| ---------------------------------- | -------------------------------------------- | ---------: |
| ローカルWSLでEDA・CV・run logging         | **Kaggle CLIでdownload**                      |        最優先 |
| Code Competition / Notebook提出の再現確認 | **Kaggle Notebook内の `/kaggle/input/...` 参照** | ROGIIでは必須級 |

黒澤さんの運用では、**ローカルCLI取得で開発 → Kaggle Notebook pathに差し替えて提出確認** が正解です。

---

## 1. コンペslug確定

| コンペ                                 | Kaggle slug                         | 用途          |
| ----------------------------------- | ----------------------------------- | ----------- |
| ROGII - Wellbore Geology Prediction | `rogii-wellbore-geology-prediction` | 本命          |
| Predicting Stellar Class            | `playground-series-s6e6`            | 練習・提出テンプレ作成 |

ROGIIの公式ページURL上のslugは `rogii-wellbore-geology-prediction`、Stellar Classは `playground-series-s6e6` です。ROGIIのデータは水平坑井軌跡と縦井戸Typewellログを含む構造で、Stellar Classは `train.csv` / `test.csv` / `sample_submission.csv` の通常Playground型です。([Kaggle][1])

---

## 2. 方法A：ローカルWSLでKaggle CLI取得

Kaggle CLIは、コンペファイルの一覧確認に `kaggle competitions files`、取得に `kaggle competitions download` を使います。公式CLIドキュメントでも、competition URL suffix、つまりslugを指定してファイル一覧・ダウンロードする形です。([GitHub][2])

### 初回セットアップ

```bash
mkdir -p ~/kaggle-bronze
cd ~/kaggle-bronze

uv venv .venv
source .venv/bin/activate

uv pip install kaggle pandas numpy scikit-learn lightgbm catboost xgboost optuna duckdb pyarrow
```

Kaggle CLIはPython 3.11+と `pip install kaggle` が前提です。認証は `kaggle auth login`、環境変数、API token file、または従来の `~/.kaggle/kaggle.json` が使えます。([GitHub][3])

従来方式なら：

```bash
mkdir -p ~/.kaggle
# KaggleのAccount/API画面から落とした kaggle.json をここへ置く
chmod 600 ~/.kaggle/kaggle.json
```

注意点として、CLIでデータ取得する前に、Kaggle側で **Join Competition / ルール承諾** が必要です。公式チュートリアルでも、データdownloadやsubmit前にコンペへ参加しルールを承諾する必要があると明記されています。([GitHub][4])

## 3. ROGIIのデータ取得

### ファイル一覧確認

```bash
mkdir -p data/raw/rogii

kaggle competitions files -c rogii-wellbore-geology-prediction
```

ROGIIは通常の `train.csv` 一枚型ではなく、`train/`、`test/`、`sample_submission.csv`、説明資料などを含む構造です。Kaggle上のNotebook input情報では、ROGII inputは約1.33GBで、`train`、`test`、`AI_wellbore_geology_prediction_task_en.pptx`、`sample_submission.csv` が確認されています。([Kaggle][5])

### ダウンロード

```bash
kaggle competitions download -c rogii-wellbore-geology-prediction -p data/raw/rogii --force

unzip -o data/raw/rogii/rogii-wellbore-geology-prediction.zip -d data/raw/rogii
```

### 想定パス

```text
data/raw/rogii/
  train/
  test/
  sample_submission.csv
  AI_wellbore_geology_prediction_task_en.pptx
```

ROGIIでは、公開されている `test/` は提出コード作成用の小さな例で、本番採点時には隠しtestに差し替わる点が重要です。公式dataページの検索結果でも、見えている `test/` はtraining set由来の少数例で、提出作成補助用と説明されています。([Kaggle][6])

## 5. 方法B：Kaggle Notebook内で取得済みdataを読む

Kaggle Notebookでは、コンペデータは通常 `/kaggle/input/competitions/<slug>/` にマウントされます。Stellarの公開Notebookでは `/kaggle/input/competitions/playground-series-s6e6/train.csv` などを読む例が確認できます。([Kaggle][8])

### Stellar Class

```python
from pathlib import Path
import pandas as pd

DATA_ROOT = Path("/kaggle/input/competitions/playground-series-s6e6")

train = pd.read_csv(DATA_ROOT / "train.csv")
test = pd.read_csv(DATA_ROOT / "test.csv")
sample = pd.read_csv(DATA_ROOT / "sample_submission.csv")

print(train.shape, test.shape, sample.shape)
```

### ROGII

```python
from pathlib import Path
import pandas as pd

DATA_ROOT = Path("/kaggle/input/competitions/rogii-wellbore-geology-prediction")

sample = pd.read_csv(DATA_ROOT / "sample_submission.csv")

train_dir = DATA_ROOT / "train"
test_dir = DATA_ROOT / "test"

print(train_dir.exists(), test_dir.exists(), sample.shape)
print(list(train_dir.glob("*"))[:5])
print(list(test_dir.glob("*"))[:5])
```

ROGIIはCode Competition寄りなので、最終的にはNotebookが `/kaggle/working/submission.csv` を出す形に合わせます。Kaggle CLIの公式チュートリアルでも、Code CompetitionではNotebookを作成し、notebook versionと出力ファイルを指定してsubmitする流れが示されています。([GitHub][4])

## 6. 黒澤さん向けの実務判断

### Stellar Class

**ローカルCLI取得だけで十分です。**

理由は、通常の `train.csv / test.csv / sample_submission.csv` 型なので、WSLで完結しやすいからです。目的はメダルではなく、48時間提出テンプレ、SQLite run log、LightGBM/CatBoost/XGBoost平均の型作りです。

### ROGII

**ローカルCLI取得 + Kaggle Notebook内path確認の両方が必要です。**

理由は、ROGIIでは見えているtestが本番testではなく、Notebook実行時に差し替わる構造を前提にすべきだからです。ローカルだけで完成させると、提出時にpath・hidden test・ファイル構造で破綻する可能性があります。

## 7. 推奨ディレクトリ

```text
kaggle-bronze/
  data/
    raw/
      stellar/
      rogii/
  notebooks/
    stellar_baseline.ipynb
    rogii_baseline.ipynb
  src/
    paths.py
    logging_sqlite.py
    metrics.py
  runs/
    kaggle_runs.sqlite
  submissions/
    stellar/
    rogii/
```

`src/paths.py` はローカルとKaggle Notebookの両対応にします。

```python
from pathlib import Path

def get_data_root(slug: str, local_root: str = "data/raw") -> Path:
    kaggle_root = Path(f"/kaggle/input/competitions/{slug}")
    if kaggle_root.exists():
        return kaggle_root
    return Path(local_root) / ("rogii" if "rogii" in slug else "stellar")
```

---

## 8. 破綻条件

| 破綻                   | 原因                                                                 | 対応                                       |
| -------------------- | ------------------------------------------------------------------ | ---------------------------------------- |
| `403 Forbidden`      | Join Competition / Rules承諾未完了                                      | Kaggle画面でJoinしてから再実行                     |
| `kaggle.json` 認証エラー  | token配置・権限ミス                                                       | `~/.kaggle/kaggle.json` と `chmod 600` 確認 |
| ROGIIでlocal testだけ良い | 公開testが小さな例で本番差し替え                                                 | Notebook提出前提で書く                          |
| Notebookでpathが違う     | `/kaggle/input/<dataset>` と `/kaggle/input/competitions/<slug>` の差 | path探索fallbackを入れる                       |
| ROGIIでsubmission列不一致 | sample_submission形式未確認                                             | 必ず `sample_submission.csv` を起点に作る        |

## 9. すぐ実行するならこの順

```bash
# 1. Stellarで練習用データ取得
mkdir -p ~/kaggle-bronze/data/raw/stellar
cd ~/kaggle-bronze
kaggle competitions download -c playground-series-s6e6 -p data/raw/stellar --force
unzip -o data/raw/stellar/playground-series-s6e6.zip -d data/raw/stellar

# 2. ROGIIで本命データ取得
mkdir -p data/raw/rogii
kaggle competitions download -c rogii-wellbore-geology-prediction -p data/raw/rogii --force
unzip -o data/raw/rogii/rogii-wellbore-geology-prediction.zip -d data/raw/rogii

# 3. 取得確認
find data/raw/stellar -maxdepth 1 -type f
find data/raw/rogii -maxdepth 2 | head -50
```

実務判断としては、**StellarはCLI取得で型作り、ROGIIはCLI取得後すぐNotebook path互換を作る**。
この順にすると、ROGIIで一番怖い「ローカルでは動くが、Kaggle提出で壊れる」を先に潰せます。
