# Superseded: 汎用 LightGBM パイプライン 土台構築

> このタスクは初期ローカル版の完了メモ。現行の正規経路は `configs/` + `runner.experiment.train` + GCS / BigQuery / Vertex。SQLite や `src/train.py` 前提の記述を新規実装の根拠にしない。

## 目的

コンペが変わっても `config.py` の 3 行を変えるだけで転用できる  
最小 LightGBM パイプラインを作る。  
初回動作確認データセットは **sklearn 内蔵 California Housing**（API 不要・即実行可）。

## 成功条件

- `make run` で California Housing の 5-fold CV RMSE が標準出力に出る
- 実験結果が BigQuery `<bqDataset>.experiments` に記録される
- 別コンペに転用する際は `configs/<competition>_*.yaml` と `env/config.yaml` を更新する

## 作成ファイル一覧

```
requirements.txt       ← uv pip install -r で一発セットアップ
src/config.py          ← TARGET / OBJECTIVE / METRIC / N_FOLDS / SEED
src/preprocess.py      ← データロード + null 埋め + OrdinalEncoding
src/runner/experiment/train.py ← LightGBM 5-fold CV ループ
src/utils/logger.py            ← BigQuery に cv_score / params を記録
src/predict.py         ← submission.csv 生成（Kaggle 提出用）
Makefile               ← setup / run を実装
```

## コンペ転用時の変更箇所（config.py のみ）

```python
TARGET    = "MedHouseVal"   # ← コンペの目的変数名に変える
ID_COL    = None            # ← コンペの ID 列名に変える（なければ None）
OBJECTIVE = "regression"    # ← binary / multiclass / regression
METRIC    = "rmse"          # ← auc / logloss / rmse / mape 等
```
